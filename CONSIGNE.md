# PROTOCOLE DE WORKFLOW AMÉLIORÉ - GEMINI & CHRISTOPHE MILLOT

## 1. RÈGLE D'OR : ZÉRO CHIRURGIE (MAJ 16/01/2026)
- **Fichiers Complets uniquement** : Pour garantir la stabilité du code et éviter de rompre l'équilibre des fichiers, je ne dois plus envoyer de blocs isolés (ex: un seul <article>).
- **Copier-Coller Intégral** : Je dois fournir le code source complet du fichier concerné (ex: uxui.php) pour que l'intégration soit un remplacement total et sans erreur.
- **Interdiction des fragments** : Toute réponse contenant des morceaux de code ou des fragments isolés est proscrite.

## 2. RÈGLE DE PRIORITÉ : STRUCTURE AVANT TEXTURE
- **Solidité du CSS** : Pour tout motif à géométrie répétitive (briques, carrelages, grilles), je dois impérativement utiliser le CSS pur pour valider la structure avant d'envisager une texture SVG.
- **Changement de Méthode** : Si un rendu visuel échoue deux fois, je dois changer radicalement de méthode technologique (passer du SVG au CSS ou inversement) au lieu de persister dans la même voie.

## 3. INTÉGRITÉ ET ACCORDS PRÉALABLES
- **Accord Explicite** : Je dois demander ton accord avant toute modification de couleurs, de classes CSS ou de l'organisation des fichiers.
- **Respect de la Structure** : Strict respect des priorités : Reset Vertical, Harmonisation des espaces (60px), et Master-Page Cloning.

## 4. PROCÉDURE DE SAUVEGARDE (GIT)
À chaque demande de mise à jour du dépôt ("push"), je dois rappeler la procédure exacte :
1. `git add .`
2. `git commit -m "[message descriptif]"`
3. `git push`
